Product Improvements for HarmonyForge
Clarify Generative Terminology: The user expressed immediate confusion over configuration settings, stating they did not understand terms like "portal, mixed and flowing" or descriptions regarding "backing chords". You must implement tooltips or translate these parameters into clearer, layman-accessible musical outcomes.
Enhance Navigational Discoverability: The user became trapped in the generation screen, asking, "How do you go back to..." before the moderator had to explicitly direct them to the "document tab". Renaming this tab to something actionable like "Edit Configuration" or "Settings" will reduce navigational friction.
Improve Interactive Affordances: A major gap in the UI is the lack of visual cues indicating that the user has edit-authority; the participant did not know they could manipulate the score until the moderator explicitly told them to click and drag notes. You need to implement cursor changes, hover states, or a brief interactive onboarding overlay to communicate this capability naturally.
Resolve Layout and Scrolling Conflicts: The participant experienced UI layout issues, noting, "I guess you can't scroll down with this [Theory Inspector] open". You must refine the responsive design so that pedagogical overlays do not obstruct the primary workspace.
Optimize System Responsiveness: The user encountered a glitch where clicking a note failed to trigger an action, stating, "I don't think it rendered... or it did not work here". Ensuring immediate, tactile feedback for every interaction is critical for maintaining user trust in the system.
